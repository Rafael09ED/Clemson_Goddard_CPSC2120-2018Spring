CPSC 212 Mystery Orders Lab

In this lab you will measure the run times of five different programs and estimate their time complexities.

To speed up things, the executable just returns a number stating the 'time' it takes. Copy the java class file TimeSink.class. It takes two arguments: a number between 1 and 5, and a value for N. For example, 

java TimeSink 3 55
There is a built-in 'time limit' so that the program will 'abort' for values that are too large.
  1. Create a table with times for each program, say with values of N being 4, 6, 10, 20 50, 100, 200. Fill in most of the table.

  2. Determine the time complexity of each program. Give brief justification.

Hints for determining the complexity of a program

  • The complexities have been chosen from the following list of functions of N: constant, log N, N, N log N, N^2, N^3, 2^N, fibonacci(N), factorial(N)
  • Using a graphing application to plot the data points might help.
  • Note that time measurements will vary slightly from one run to the next. You might average a few runs with the same input.
  • As you look for a pattern, look at the larger times first.
  • In each program, there is a certain amount of overhead built in. This overhead is the same regardless of the input parameter. You might want to subtract the shortest run time from the other runs to try to eliminate this.
  • If necessary, run the program for a few of the intermediate numbers.
